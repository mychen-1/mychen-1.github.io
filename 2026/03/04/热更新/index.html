热更新

发表于 2026-03-04 分类于 Erlang

概述

Erlang 热更新机制允许在不停机的情况下更新运行中的代码。

1. Erlang 中的代码版本状态

代码存在多种版本状态：

  • current：当前版本
  • old：旧版本
  • new：新版本
  • pending delete：待删除版本

2. 各版本的切换流程

  1. 进程首次加载时：current 只有一份版本
  2. 进行热更新时：存在 2 份版本，current > old，以及一份 new
  3. 再次更新时：检测 old 版本是否还有进程调用
    • 若有：old → pending delete
    • 然后：new → old，并生成新的 new
  4. 再次更新时：pending delete 会被强制删除，使用该版本的进程会收到 {exit, pid, killed} 消息

3. 热更新触发时机

  1. 外部调用：例如 lists:keyfind 时，会检测 lists 模块版本
  2. receive 接收消息时：检测自身进程的代码版本

4. 热更新失败场景

  1. 进程未切换：进程一直执行尾递归，无法进入检测点
  2. 进程状态不兼容：state 结构变化，会报错直至 timeout，然后触发进程重启策略
  3. 清除旧代码错误
  4. 原子使用上限等限制

5. 更新接口

接口 说明
code:soft_purge(Mod) 安全热更新。若仍有旧进程调用，会等待完成
code:purge(Mod) 强制热更新。会杀掉使用旧代码的进程
code:atomic_load(Mods) 以原子形式更新多个模块，防止单个模块更新导致不一致
