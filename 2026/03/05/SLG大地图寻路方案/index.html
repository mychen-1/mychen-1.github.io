SLG 大地图寻路方案——A*、漏斗与路径广播

发表于 2026-03-05 分类于 游戏开发

概述

SLG 大地图寻路涉及大规模、不规则地形的路径计算与同步，核心流程可分为三步：A 算区域* → 漏斗算法拉直 → 路径广播。该流程要点。

一、整体流程

二、第一步：A* 算法——算出经过的区域

2.1 作用

在网格或导航网格（NavMesh）上，从起点到终点搜索经过的三角形/区域序列，为后续漏斗算法提供「通道」。

2.2 示意要点

  • 起点（绿格）、终点（红格）
  • 障碍（蓝格）不可通行
  • 每个格子记录 G 成本（已走距离）和 H 估值（到终点距离），箭头表示父节点，用于回溯路径
  • 通过 Open List / Closed List 逐步扩展，得到一条从起点到终点的区域序列

2.3 输出

一条三角形列表（或区域列表），表示从起点到终点需要经过的可通行区域，作为漏斗算法的输入。

三、第二步：漏斗算法——拉直路径

3.1 作用

A* 得到的路径往往曲折，漏斗算法在同一通道内将路径拉直，得到更短、更自然的行走路径。

3.2 输入与输出

  • 输入：A* 生成的三角形列表，起点、终点（图中蓝点）
  • 输出：从起点到终点的最短折线路径（图中绿色粗线）

3.3 算法步骤

  1. 邻接边：计算三角形列表中的邻接边（两三角形共用边）
  2. 构造漏斗：从起点开始，以第一条邻接边为底构造漏斗
  3. 迭代收紧：
    • 将左右漏斗边的终点移动到下一条邻接边
    • 若漏斗变窄或不变 → 移动有效
    • 若漏斗变宽 → 该边回退，移动无效
  4. 特殊情况：漏斗口角度变负（原右边转到左边）时，被盖过的边终点作为路径点，漏斗起点移到该点，用新三角形出边重新构造漏斗
  5. 结束：到达最后一条邻接边，漏斗口收缩到终点，算法结束

3.4 有效性判断

  • 左/右边分别考虑角度变化
  • 漏斗变窄或不变 → 有效
  • 漏斗变宽 → 无效，需回退

五、地编简述

地编负责地图的编辑与生产。典型 SLG 大地图特征例如：

  • 规模：7680×7680 米，9-3-1 结构
  • 容量：7000+ 玩家主堡、7000+ 怪物、4 万+ 矿点

地编产出可通行区域、障碍、导航网格等，为 A* 和漏斗算法提供底层数据。寻路与广播逻辑建立在地编数据之上。

六、小结

步骤 算法 输入 输出
1 A* 起点、终点、可通行区域 经过的三角形/区域列表
2 漏斗算法 三角形列表、起点、终点 拉直后的最短路径
3 路径广播 路径线段、玩家视野 视野内玩家的路径同步
