操作系统基本介绍

发表于 2026-02-08 分类于 操作系统

概述

该篇主要介紹操作系統

硬盘扇区

我们都知道 硬盘存储数据是以扇区为单位， 一个扇区 512字节。 一个盘面分为2面。

假设盘面磁道65536， 每个磁道 1024 扇区， 这样一个盘面大小 2 * 65536 * 1024 * 512 = 64 G。

面对扇区划分， 盘面是一个同心圆， 每个磁道周长不一样， 假设每个磁道扇区一样， 这样肯定是内部紧密外部稀疏， 不能接收， 外面浪费空间。 假设不同磁道扇区不同，又不好计算。 所以从0编号开始 一直递增到最后一个扇区， 这个叫做逻辑扇区号。

分段分页

分段：

面对进程需要使用多大内存， 如何获取， 从内存到磁盘。 假设A B C三个进程需要 10M 100M 20M 空间， 面对128M内存， 只能同时运行A B，加载到内存中，同时 A 0 -10M的地址， B 10 -110M的地址。 映射到磁盘也是这样一段一段的。

分页：

中间加了一次虚拟地址， 程序不需要关注内存分配。 程序叫 虚拟页， 内存叫物理页。 磁盘叫磁盘页。

映射使用MMU部件 CPU -> 虚拟地址 -> MMU -> 物理页 （找到就返回，没找到下一步）-> 磁盘页。

线程调度和 优先级

1.以单处理器为例， 在多线程模式下， 不断切换线程叫做 线程调度。

2.线程有三种状态： 运行， 就绪， 等待。

3.每个线程运行的时间 叫做时间片

4.每个线程都有优先级， 用来决定cpu执行哪个。

5.频繁等待的线程叫做 I/0密集型线程， 很少等待的叫做CPU密集线程。 cpu更加喜欢 I/O密集型的，所以更愿意分配给他们。

6.在优先级调度下， 情况叫做饿死， 低优先级得不到调用， 后续会将该线程提高级别

7.不可抢占线程， 意思如字面，除非让出时间片 或 自己进入等待状态， 不然一直占用。

写时复制

为什么fork创建新的线程很快， 因为创建新的线程， 使用内存还是用旧线程（即创建线程）， 所以没什么开销， 当新旧线程 有写操作修改时，才会复制 开辟新空间，这个操作叫做写时复制。

杂乱无章

  1. 程序源代码被编译后分为2种段， 程序指令和程序数据，代码段属于程序指令，数据段 和 .bbs属于程序数据。 原因1：cpu L1 cache 分为数据缓存和指令缓存关联。 原因2： 指令只需要读， 而数据 需要读写 原因3：共享， 当系统运行着多个该程序副本时，指令都是一样的
