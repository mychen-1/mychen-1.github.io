tcp,http,websocket差异

发表于 2026-02-08 分类于 计算机网络

概述

该篇主要介绍tcp,http,websocket差异。

1.网络层面

tcp属于传输层， http和websocket属于应用层。

TCP

对于我们开发者来说，其实不需要在意tcp底层如何实现，只需要知道TCP的数据是连贯的可信的。

为了实现可靠性： TCP有三次握手和四次挥手， 其实为了保证 端对端连接成功，

介绍一下ACK：简单理解就是： 每次传输都会携带该值，接受成功后将ACK+1， 这样当出现丢包的时候， 发现ACK不正确，这样就可以采用重传机制来确保数据正确。

另外TCP还有其他功能：

流量控制 (Flow Control)：通过滑动窗口 (Sliding Window) 机制，接收方可以告知发送方自己还有多少缓冲区空间可以接收数据，防止发送方过快发送数据导致接收方缓冲区溢出。

拥塞控制 (Congestion Control)：通过慢启动、拥塞避免等算法，TCP 能够感知网络拥堵状况，动态调整发送速率，避免造成网络瘫痪。

HTTP

HTTP建立在TCP之上，服务器和客户端之间有一套规则（规则目的|业务层可以自我增强安全性）。

特性：

1.请求-响应： 通信只能客户端发起，服务端返回，不能服务端推送。

2.无状态:每次请求都是独立的。

HTTP1.0：短连接

每次HTTP都需要经历完成的周期，对于固定端对端来说，无疑是格外的开销

HTTP1.1：长连接:

当客户端-服务端完成一次请求后，不会立即关闭TCP， 保持一段时间。其中会使用心跳机制， 防止僵尸客户端。

问题：当需要服务器主动推送，怎么实现？

由此诞生websocket

websocket

也是建立在TCP之上，对比于HTTP优势有：

1.开销更少： websocket的数据帧更小。

2.全双工通信：服务端可以向客户端主动推送消息。

3.持久化。

为什么websocket可以向客户端主动推送，http不行呢？

因为websocket的数据帧可以自己维护。HTTP设计理念不支持。
