7.Core Erlang (Erlang 中间语言）

查看方式：c(mod, to_coreo)

特点：

•简化版 Erlang，去掉所有语法糖
•明确作用域、变量綁定和函数调用方式
•统一的控制流表示
设计目标
•简化编译器后端：统一的中间表示
•优化友好：便于进行各种代码优化
•跨平台支持：可转换为不同目标平台代码

8.Core Pass（中间语言优化）

查看方式：c(mod, to_core)

职责：对 Core Erlang 进行一系列分析和优化

优化功能

1.死代码刪除：移除不可达代码

2.表达式简化：常量折疊、算术优化
3.变量绑定优化：消除无意义的变量绑定

优化模块

•sys-core_fold：常量折叠和表达式简化
• sys_core_ bsm：二进制操作优化
• sys_core_dsm：数据结构优化
•sys_core_inline：函数内联
• sys_core_alias：别名消除
• v3_core：核心优化器

性能影响

•编译时间：增加 10-20% 编译时间
•运行时性能：提升 5-15% 执行效率
.代码大小：通常减少 10-30% BEAM 文件大小

9.Kernel Erlang + Kernel Pass

查看方式：cqod, to_kernel)
职责：将Erlang 中间语言向机器层中间语言过渡

核心转换

  1. 显式化控制流：使用 do / goto结构
  2. 寄存器分配准备：为后续代码生成做准备
  3. 异常处理优化：统一异常处理机制

10.ASM Pass

作用：从“虚拟机指令“到”机器指令”的桥梁
核心任务：奇存器分配与指令选择
扩展：编译器可以被分为前端和后端。
编译器前端负责产生机器无养的中间代码,编译器后端将中间代码转换成Erlang对标机器代码。这样对于一些可以跨平台的编洋器而言,它们可以针对不同的平台使用同一个前端和针对不同机器平台的数个后端

11.BEAM 汇编（S文件）

查看方式：c（Mod，’S’） 或编译时生成-s 文件

核心概念

模块结构
. module ： 模块名
• exports：导出函数列表
• labels：标签总数（用于跳转）
•label：标签定义（代码位置标记）

函数定义

•allocate：分配堆和栈空间
(allocate, Stackwords, Heapwords}
•deallocate：销毁栈空间

(call-ext, Arity, textfunc, Module, Function, Arity}) %外部调用

12.链接

1.链接分为静态链接和动态链接。

静态链接：
编泽器和链接器（Id）在生成可执行文件时，会把 a.o和 b.o拼死在一起。如果 a.o里调用了 b.o的函数，链接器会计算好内存地址偏移量，把它们’焊死”。程序跑起来时，CPU 直接跳到那个算好的地址去。
动态链接：
Erlang 编译器生成 a,beam 和 b.beam 时，它们互不认识。
当a 模块运行到调用 b:foo() 这一行时，虛似机(VN）会暂停一下，问：“谁是b？b在哪？b加载了吗？然后现场把它们连起来。

2.Erlang 链接的两个关键表：导出表与导入表

我们在上一个问题中用 beam_lib:info（）。 看到的chunk 此时就派上用场了：

• ExpT (Export Table)

：我对外提供了什么？
。例如 Tists.beam 的ExpT 里写着：我有 reverse/1. map/2。

• ImpT (Import Table)

：我需要谁？
。例如my-code.beam 的ImpT 里写着：我需要 lists:reverse/1。
链接过程就是 VM 拿着 ImPT 去匹配别人 ExpT 的过程。

3.Erlang 链接的三种模式（由慢到快）

Erlang 的链接机制非常智能，它分为几个阶段，目的是为了支持热代码更新 (Hot Code Loading)

阶段一：完全动态 (Generic Instruction)

当模块刚加载，或者代码还没被频繁执行时。
•指令：call_ext (Call External)
•行为：VM 看到 lists：reverse(L），它会去一个全局的哈希表 (Export Table）里查 lists 模块的reverse 函数的地址
。特点：最慢，但最灵活。因为每次调用都要查表，所以如果你热更新了 Iists模块，下一次查表自然就查到新代码了。

阶段二：修补 (Patching / Back-patching)

Erlang VM 发现你老是调这个函数，它觉得每次查表太累了，就会修改字节码。
。行为：VM 会把那条 call-ext指令直接改亏成一条指向目标函数内存地址的直接跳转指令。
。特点：这就变成了类似C 语言的函数指针调用，速度非常快。

阶段三：本地调用优化 (Local Call)

如果是同一个模块内部调用（比如 a函数调 b 函数），编泽器在编泽成.beam 时就已经算好相对偏移量了不需要查表，直接跳。
题外话：动态链接是实现热更新的关键。
